Справка: что означают метрики на графиках и в таблице
Метрика Что показывает Как интерпретировать
TTFT Время от отправки запроса до первого фрагмента текста в стриминговом чате (time to first token). Меньше — лучше
Total Полная длительность одного вызова API — от старта до готового ответа. Меньше — лучше
e2e t/s Пропускная способность «на весь запрос»: токены выхода (или оценка) делённое на полное время. Больше — лучше
gen t/s Скорость фазы генерации после первого токена (стриминг). Для нестрима часто нет смысла. Больше — лучше
prompt Размер входа в токенах (из API usage или оценка по длине текста). Не «лучше/хуже» — зависит от промпта и задачи
compl Токены на выходе; для транскрипции часто оценка по длине расшифровки. Нейтрально — длина ответа не равна качеству
chars Для чата — символы в ответе; для эмбеддинга — размерность вектора (не текст); для реранка — число позиций в ответе. Смотрите тип замера в колонке «Тип»
chunks Сколько стриминговых чанков принесли непустой текст (гранулярность потока). Нейтрально — не метрика качества
Δ mean / Δ max Средняя и максимальная пауза между чанками с текстом в стриме (секунды). Меньше — лучше (ровнее поток)
N Сколько записей замеров попало в выбранное окно времени. Больше — устойчивее статистика
Err % Доля замеров с ошибкой (HTTP/таймаут/исключение) среди всех записей в окне. Меньше — лучше
стрим / usage Доля успешных прогонов в стриминговом режиме; доля, где токены пришли из поля usage API. Справочно, для контекста сравнения

Задержка до первого токена в стриминговом чате. Для блокирующего режима и нечатовых целей обычно нет точки. Меньше — лучше.

Полное время одного запроса к провайдеру. Меньше — лучше.

Сквозная скорость: токены выхода на секунду за весь цикл запроса. Больше — лучше.

Скорость генерации после появления первого токена (стрим). Больше — лучше. Для нестрима часто пусто.

Объём входа в токенах. Не показатель «лучше/хуже» — зависит от промпта; сравнивайте при той же задаче.

Объём выхода в токенах (или оценка). Нейтрально — длина ответа не равна качеству модели.

Для чата — символы ответа; для эмбеддинга здесь размерность вектора; для реранка — число результатов. Смотрите колонку «Тип» в сводке.

Сколько чанков стрима содержали текст. Нейтрально — про дробность потока, не про качество.

Средняя пауза между соседними «текстовыми» чанками в стриме. Меньше — лучше (плавнее отдача).

Самая длинная пауза между чанками — индикатор «заикания» потока. Меньше — лучше.
